## Lab.3: Умножение матриц (MPI)

### 1. Задание
Модифицировать программу из лабораторной работы №1 для параллельной работы с использованием технологии MPI. Провести эксперименты с разными размерами матриц (200, 400, 800, 1200, 1600, 2000) и разным количеством процессов (1, 2, 4, 8).

### 2. Программная реализация
`main_mpi.cpp` – основная программа на C++ с использованием MPI.

Параллелизация реализована путем разделения строк матрицы A между процессами. Каждый процесс вычисляет свою часть результирующей матрицы C, после чего результаты собираются на главном процессе. Матрица B рассылается всем процессам с помощью `MPI_Bcast`. Для обмена данными используются функции `MPI_Send` и `MPI_Recv`.

### 3. Алгоритм работы
- Процесс 0 читает матрицы A и B из файлов
- Матрица B рассылается всем процессам
- Строки матрицы A распределяются между процессами
- Каждый процесс вычисляет свою часть произведения
- Результаты отправляются процессу 0 и сохраняются в файл

### 4. Результаты экспериментов

**Время выполнения (секунды)**

| Размер | NP=1 | NP=2 | NP=4 | NP=8 |
|--------|------|------|------|------|

| 400 | 0.6935 | 0.3509 | 0.4358 | 0.1735 |
| 800 | 5.1571 | 2.1654 | 1.8036 | 0.9156 |
| 1200 | 17.6876 | 10.6146 | 5.9585 | 2.8292 |
| 1600 | 54.4936 | 28.7965 | 14.4045 | 7.2385 |
| 2000 | 57.4872 | 39.0341 | 27.3586 | 12.1357 |

**Эффективность параллелизации**

| Размер | NP=2 | NP=4 | NP=8 |
|--------|------|------|------|
| 200 | 0.90 | 0.95 | 0.99 |
| 400 | 0.99 | 0.40 | 0.50 |
| 800 | 1.19 | 0.72 | 0.70 |
| 1200 | 0.84 | 0.74 | 0.78 |
| 1600 | 0.95 | 0.95 | 0.94 |
| 2000 | 0.74 | 0.53 | 0.59 |

![](lab3/time.png)

### 5. Верификация
Для всех размеров и количества процессов результаты программы совпали с вычислениями NumPy. Максимальное расхождение составило менее 10⁻⁸.

### 6. Анализ результатов
- **Маленькие матрицы (200×200):** MPI показывает ухудшение производительности (ускорение 0.46-0.76x)
- **Средние матрицы (400×400):** Начинает проявляться положительный эффект (ускорение до 2.38x)
- **Большие матрицы (800-2000):** Ускорение растет (до 3.88x при 8 процессах)

### 7. Вывод
Программа успешно модифицирована для параллельной работы с MPI. Для матрицы 2000×2000 время сократилось с 27.65 до 7.12 секунд (ускорение 3.88x). Наилучшая эффективность (83-96%) достигается при использовании 2-4 процессов для больших матриц.

## Lab.4: Шифрование квадратом Полибия (CUDA)

### 1. Задание
Модифицировать программу шифрования квадратом Полибия из лабораторной работы №1 для параллельной работы с использованием технологии CUDA. Провести эксперименты с разными размерами входных данных и различными конфигурациями сетки блоков/нитей.

### 2. Программная реализация
`polybius_cuda.cu` – основная программа на C++ с использованием CUDA.

Параллелизация реализована путем распределения символов текста между нитями GPU. Каждая нить обрабатывает один символ: читает его из глобальной памяти, находит координаты в квадрате Полибия и записывает результат (пару цифр) в выходной массив.

Квадрат Полибия хранится в постоянной памяти GPU (`__constant__`). Верификация выполняется скриптом `test.py`.
